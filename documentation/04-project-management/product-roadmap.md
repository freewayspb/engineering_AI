## 2. Фаза 1: Внутренняя разработка (MVP для инженерной организации)

### 2.1 Этап 1: Переход от прототипов к продукту (1 месяц)
**Цель**: Адаптация прототипов для продукта, настройка инфраструктуры разработки, формирование команды

**Задачи**:
- Анализ результатов прототипирования и определение приоритетов для продукта
- Адаптация архитектуры прототипов под масштабируемое решение
- Настройка инфраструктуры разработки (CI/CD, тестовое окружение, репозитории)
- Онбординг команды разработки и передача знаний от прототипов
- Планирование спринтов и определение MVP для внутреннего использования
- Подготовка технической базы (Docker окружения, базы данных, векторные БД)

**Трудозатраты**:
- PM: 80 часов (координация, планирование спринтов)
- Analyst: 80 часов (анализ прототипов, приоритизация функций)
- Architect: 160 часов (адаптация архитектуры, проектирование инфраструктуры)
- Backend Dev: 160 часов (2 чел × 80 ч: настройка инфраструктуры, изучение прототипов)
- Frontend Dev: 160 часов (2 чел × 80 ч: настройка окружения, изучение прототипов)

**Стоимость**: ~1,190,000 руб

**Результаты**:
- Адаптированная архитектура продукта на основе прототипов
- Настроенная инфраструктура разработки (CI/CD, тесты, окружения)
- Онбординг команды и передача знаний
- План разработки MVP с приоритизацией функций
- Техническая база для начала разработки

## 5. Риски и митигации

### 5.1 Технические риски

**Риск**: Сложность интеграции с различными форматами документов  
**Вероятность**: Средняя  
**Влияние**: Высокое  
**Митигация**: Использование проверенных библиотек, раннее прототипирование сложных форматов

**Риск**: Производительность LLM на локальном оборудовании  
**Вероятность**: Средняя  
**Влияние**: Среднее  
**Митигация**: Оптимизация моделей, кэширование, возможность использования GPU

**Риск**: Качество извлечения данных из документов  
**Вероятность**: Средняя  
**Влияние**: Высокое  
**Митигация**: Итеративное улучшение промптов, валидация с экспертами

### 5.2 Бизнес-риски

**Риск**: Изменение требований в процессе разработки  
**Вероятность**: Высокая  
**Влияние**: Среднее  
**Митигация**: Гибкая методология разработки, регулярные демо, управление изменениями

**Риск**: Задержки в получении обратной связи от внутренних пользователей  
**Вероятность**: Средняя  
**Влияние**: Среднее  
**Митигация**: Регулярные демо, выделение времени на обратную связь

**Риск**: Сложность выхода на рынок  
**Вероятность**: Средняя  
**Влияние**: Высокое  
**Митигация**: Ранний анализ рынка, пилотные проекты с потенциальными клиентами

## 8. Оптимизация бюджета (опционально)

### 8.1 Вариант с сокращенной командой
- Уменьшение команды на 1-2 человека на некоторых этапах
- Использование аутсорсинга для отдельных задач
- **Экономия**: ~2,000,000 - 3,000,000 руб
- **Риск**: Увеличение сроков на 1-2 месяца

### 8.2 Вариант с поэтапным финансированием
- Разработка по этапам с финансированием каждого этапа отдельно
- Возможность корректировки планов между этапами
- **Преимущество**: Снижение финансовых рисков

### 8.3 Вариант с MVP для рынка
- Сокращение Фазы 1 до минимального MVP (6 месяцев)
- Быстрый выход на рынок с базовым функционалом
- **Экономия**: ~3,000,000 руб
- **Риск**: Необходимость доработок после выхода на рынок
