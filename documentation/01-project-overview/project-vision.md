## 1. Обзор проекта

### Название проекта
AI GOST - интеллектуальная система обучения на корпоративных документах с возможностью текстовых запросов

### Краткое описание
Проект направлен на создание интеллектуальной системы, которая "учится" на корпоративных документах и отвечает на текстовые запросы пользователей. Система работает в проектно-ориентированном контексте, позволяя извлекать данные, искать аналоги по проектам и формировать структурированные отчеты в Excel. Включает разработку и тестирование прототипов по ключевым направлениям: обучение системы на документах, обработка текстовых запросов, проектно-ориентированная классификация и извлечение данных.

---

## 2. Концепция интеллектуальной системы

### 2.1 Принцип работы системы
AI GOST - это интеллектуальная система, которая работает как "умный сотрудник", способный:
- **Изучать** корпоративные документы и накапливать знания
- **Понимать** контекст проектов и технических решений
- **Отвечать** на естественные текстовые запросы пользователей
- **Извлекать** нужные данные из изученных документов
- **Искать** аналоги по проектам и формировать отчеты

### 2.2 Ролевая модель
**Учитель системы**:
- Загружает проверенные документы в систему
- Обучает систему на корпоративных данных
- Валидирует качество обучения

**Пользователь системы**:
- Делает текстовые запросы ("вытащи данные про козырек")
- Получает структурированные ответы в Excel
- Ищет аналоги по проектам

### 2.3 Проектно-ориентированная работа
Все данные в системе привязаны к конкретным проектам:
- Подстанция беговая
- Козырек
- И другие проекты заказчика

Система может:
- Извлекать ведомости объемов работ по проекту
- Находить аналоги ("где у нас есть козырьки")
- Формировать комплекты рабочей документации

## 3. Бизнес-цели и ценность

### Основные бизнес-цели
1. **Создание интеллектуальной системы обучения** - система изучает корпоративные документы и накапливает знания
2. **Автоматизация извлечения данных по запросам** - пользователи могут делать текстовые запросы ("вытащи данные про козырек")
3. **Проектно-ориентированная работа** - все данные привязаны к конкретным проектам (подстанция, козырек и т.д.)
4. **Поиск аналогов по проектам** - система может найти похожие проекты и извлечь нужные данные
5. **Автоматизация формирования отчетов** - создание структурированных отчетов в Excel по запросу
6. **Интеграция с нормативной базой** - система изучает ГОСТы и нормативные документы для валидации

### Ожидаемая бизнес-ценность
- **Экономия времени**: мгновенные ответы на текстовые запросы вместо ручного поиска в документах
- **Снижение затрат**: автоматизация извлечения данных и формирования отчетов
- **Повышение качества**: система учится на проверенных документах и обеспечивает консистентность
- **Масштабирование**: накопление знаний позволяет обрабатывать новые проекты быстрее
- **Проектная эффективность**: быстрый поиск аналогов и извлечение нужных данных по проектам

## 5. Объем проекта

**Прототипирование**
- Прототип обучения системы на корпоративных документах

- Прототип проектно-ориентированной классификации документов
- Прототип извлечения данных из PDF/изображений/чертежей (OCR)
- Прототип структурирования данных в Excel формат
- Прототип поиска аналогов по проектам

**Документация**
- Техническое задание для производственной системы
- Отчет по анализу AI-моделей
- Архитектурная схема решения

- **Разработка производственной системы** - только ТЗ и прототипы

- **Интеграция с существующими системами** - только архитектурная проработка
- **Обучение пользователей** - не входит в текущий этап
- **Сопровождение и поддержка** - только передача результатов

## 6. Ключевые требования и ограничения

### Временные ограничения
- **Срок выполнения**: 30 календарных дней с момента предоплаты
- **Ключевые контрольные точки**:
  - День 14: Завершение интервью и анализа требований
  - День 21: Готовность первых прототипов
  - День 28: Завершение архитектурной проработки
  - День 30: Сдача всех результатов

### Бюджетные ограничения
- **Общий бюджет**: 615 000 рублей
- **Модель оплаты**: 50% предоплата, 50% по результатам

### Ресурсные ограничения
- **Команда проекта**: 4 специалиста (132 часа общих трудозатрат)
- **Доступность заказчика**: зависит от графика стейкхолдеров
- **Образцы данных**: качество прототипов зависит от предоставленных образцов

### Технические ограничения
- Требования к безопасности на уровне коммерческих документов (не "для служебного пользования")
- Совместимость с существующими системами
- Производительность и масштабируемость решений
- Локальное развертывание без доступа к интернету

### Нормативные ограничения
- Соответствие требованиям защиты персональных данных
- Отраслевые стандарты и нормативы
- Требования к архивированию и аудиту

## 7. Критерии успеха

### Критерии успеха проекта (текущий этап)
1. **Техническое задание** принято заказчиком без критических замечаний
2. **Все прототипы** демонстрируют техническую осуществимость задач
3. **Архитектурное решение** соответствует бизнес-требованиям и ограничениям
4. **Отчет по анализу** содержит обоснованные рекомендации по выбору технологий
5. **Проект завершен** в срок и в рамках бюджета

### Критерии успеха будущей системы
- Точность ответов на текстовые запросы: не менее 90%
- Скорость обработки запросов: не более 5 секунд
- Качество обучения системы: способность извлекать данные из новых документов без дополнительного обучения
- Проектно-ориентированная работа: корректная привязка данных к проектам
- Поиск аналогов: точность поиска похожих проектов не менее 85%
- Формирование отчетов в Excel: корректность структурирования данных не менее 95%

## 8. Основные риски

### Высокие риски
**Недоступность стейкхолдеров** для интервью в первые 2 недели
- *Влияние*: Задержка всего проекта
- *Митигация*: Раннее планирование встреч, альтернативные форматы интервью

**Отсутствие качественных образцов данных** для обучения системы
- *Влияние*: Низкое качество обучения, неточные ответы на запросы
- *Митигация*: Запрос образцов на этапе планирования, использование синтетических данных

**Сложность обучения системы** на корпоративных документах
- *Влияние*: Система не сможет корректно отвечать на текстовые запросы
- *Митигация*: Поэтапное обучение, валидация на каждом этапе

### Средние риски
**Расширение объема работ** в процессе выполнения
- *Влияние*: Превышение бюджета и сроков
- *Митигация*: Четкое управление изменениями, документирование всех запросов

**Технические ограничения** существующей инфраструктуры
- *Влияние*: Необходимость пересмотра архитектурных решений
- *Митигация*: Ранний анализ ИТ-инфраструктуры, планирование альтернатив
