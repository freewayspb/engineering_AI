Описание технического задания
AI GOST
 
Цель:
Создание Локальной информационной системы на базе принципов обработки данных с помощью искусственного интеллекта, для бизнес-процессов обособленной организации.
 
Структура построения информационной системы:
1. Формирование дерева системы на основании штатного расписания организации;
2. Фиксация ролей доступа для сотрудников:
- Администратор;
- Учитель;
- Пользователь.
3. Организация авторизации доступа сотрудников в систему, с учетом ролей;
4. Организация и настройка взаимодействия участников информационной системы (роевых сотрудников) по средству ИИ-агента.  
5. Организация принципов и механизмов экспорта информации в систему, по средству следующих типов файлов:
- Табличные данные (xlsx);
- Текстовые данные (RTF);
- Графические данные (dwg/DXF);
- Скан образы читаемы/не читаемые (pdf);
- Сметы (ARP/GSFX/xml).
6. Организация принципов и механизмов текстового запроса в систему;
7. Организация принципов и механизмов импорта файлов из системы по текстовому запросу, по средству следующих типов файлов:
- Табличные данные (xlsx);
- Текстовые данные (RTF);
- Графические данные (dwg/DXF);
- Скан образы читаемы/не читаемые
8. Организация структуры типовых отчетных данных по запросу пользователя;
9. Организация структурированного хранения базы данных на локальных серверах компании.
 
Принципы информационной системы:
1. Интерфейс системы должен отвечать «Чистого-листа».
Целью данного принципа является отрисовка/написания любого вида отчетных форм для импорта информации из системы, с минимальными программными ограничениям;
2. Система должна позволять, без привлечения высококвалифицированных АйТи инженеров (силами Администратора) производить дописание простейших чат ботов, на базе имеющегося интерфейса.
3. Система должна иметь возможность взаимодействовать с иными информационными базами данных организации, с целью экспорта информации, по принципу «Розетки»;
4. Система должна иметь возможность управляемого/ограниченного взаимодействия с сетью Ethernet, с целью экспорта информации.
5. Система должна иметь возможность настройки удаленного мобильного рабочего места, с целью оперативного получения информации;
6. Система должна воспринимать русский и английский языки.
