# Use Cases - Проект AI GOST
**Версия**: 0.1 (табличный формат для Excel/Google Sheets)  
**Дата**: 01.10.2025  

---

## Таблица Use Cases

**Формат**: ID | Название | Краткое описание | Акторы | Предусловия | Основной сценарий | Ожидаемый результат

---

### UC-01 | Извлечение текста из документов
**Краткое описание**: Пользователь загружает PDF/изображение/чертёж, система извлекает текст  
**Акторы**: Пользователь, OCR-система  
**Предусловия**: Документ доступен для загрузки, система OCR настроена  
**Основной сценарий**: Пользователь загружает файл → система автоматически определяет тип документа (PDF/изображение/DWG/смета) → запускает OCR/парсинг → обрабатывает документ → извлекает текст/данные  
**Ожидаемый результат**: Определён тип документа; распознанный текст/данные отображаются и сохраняются в структурированном виде

---

### UC-02 | Структурирование извлеченных данных
**Краткое описание**: Система преобразует извлеченные OCR/NLP данные (таблицы и текст) в структурированный формат для последующей интеграции  
**Акторы**: Пользователь, Модуль структурирования  
**Предусловия**: Данные извлечены из документов (UC-01)  
**Основной сценарий**: Анализ извлеченного контента → сопоставление полей и типов → формирование таблиц/JSON/CSV → валидация по схемам → экспорт в целевой формат (JSON/CSV/XLSX)  
**Ожидаемый результат**: Корректно структурированные данные, экспортированные в целевой формат согласно требованиям интеграции

---

### UC-03 | Тематическая классификация документов
**Краткое описание**: Система распределяет документы по темам с использованием NLP согласно набору категорий, согласованному с заказчиком  
**Акторы**: Пользователь, NLP-система  
**Предусловия**: Загружено несколько документов, NLP-модель обучена  
**Основной сценарий**: Система анализирует содержание документов → применяет NLP-модель → определяет тематические категории → группирует документы  
**Ожидаемый результат**: Документы автоматически распределены по тематическим папкам/категориям

---

### UC-04 | Анализ требований безопасности
**Краткое описание**: БА и Архитектор проверяют интеграцию с ИТ-инфраструктурой заказчика  
**Акторы**: Бизнес-аналитик, Архитектор, Представители ИТ заказчика  
**Предусловия**: Доступны данные о системах заказчика, проведены интервью  
**Основной сценарий**: БА собирает требования ИБ → Архитектор анализирует инфраструктуру → определяются точки интеграции → разрабатываются требования безопасности  
**Ожидаемый результат**: Требования безопасности документированы и учтены в архитектуре

---

### UC-05 | Создание прототипа AI-функции
**Краткое описание**: Специалист по ИИ создаёт минимальный прототип для одной задачи (OCR/NLP/классификация)  
**Акторы**: Специалист по ИИ, Бизнес-аналитик  
**Предусловия**: Есть четкое бизнес-требование, доступны образцы данных  
**Основной сценарий**: Выбор технологии → разработка минимального прототипа → тестирование на образцах → документирование результатов  
**Ожидаемый результат**: Рабочий прототип готов, протестирован и задокументирован

---

### UC-06 | Согласование архитектуры системы
**Краткое описание**: Архитектор предлагает техническую схему, заказчик утверждает решение  
**Акторы**: Архитектор, Заказчик, ИТ-департамент заказчика  
**Предусловия**: Подготовлены архитектурные материалы, проанализированы требования  
**Основной сценарий**: Презентация архитектурной схемы → обсуждение технических решений → внесение корректировок → получение одобрения  
**Ожидаемый результат**: Архитектура системы утверждена и готова для включения в ТЗ

---

### UC-07 | Подготовка технического задания
**Краткое описание**: БА компилирует финальный документ ТЗ на основе всех исследований  
**Акторы**: Бизнес-аналитик, Архитектор, Менеджер проекта  
**Предусловия**: Исследования завершены, прототипы готовы, архитектура согласована  
**Основной сценарий**: Сбор всех материалов → составление ТЗ → внутренняя проверка → презентация заказчику → внесение правок → финальное согласование  
**Ожидаемый результат**: Утвержденное техническое задание готово для следующего этапа разработки

---

### UC-08 | Исследование рынка AI-решений
**Краткое описание**: Команда анализирует доступные AI/ML технологии и выбирает оптимальные  
**Акторы**: Специалист по ИИ, Архитектор, Бизнес-аналитик  
**Предусловия**: Определены технические требования, есть доступ к информации о решениях  
**Основной сценарий**: Поиск решений → анализ по критериям (стоимость, точность, интеграция) → тестирование топ-решений → подготовка рекомендаций  
**Ожидаемый результат**: Отчет с рекомендациями по выбору технологического стека

---

### UC-09 | Обработка специализированных форматов
**Краткое описание**: Система обрабатывает чертежи DWG и сметные форматы "ARP"/"GSFX" (по Приложению 3)  
**Акторы**: Пользователь, Модуль обработки DWG/смет  
**Предусловия**: Загружены файлы DWG/ARP/GSFX, настроены соответствующие модули  
**Основной сценарий**: Загрузка файла → определение формата → извлечение целевых полей (слои/объекты для DWG; позиции/суммы для смет) → структурирование данных  
**Ожидаемый результат**: Извлеченные данные из DWG и смет структурированы для дальнейшего использования

---

### UC-10 | Валидация и тестирование прототипов
**Краткое описание**: Команда тестирует все прототипы на реальных данных заказчика  
**Акторы**: Вся команда проекта, Представители заказчика  
**Предусловия**: Прототипы готовы, предоставлены тестовые данные заказчика  
**Основной сценарий**: Подготовка тест-кейсов → запуск тестирования → анализ результатов → документирование выводов → презентация заказчику  
**Ожидаемый результат**: Все прототипы протестированы, результаты задокументированы, получена обратная связь

---
