## Трассировка BR ↔ UC

| BR (из BRD) | UC | Назначение | Примечание |
|---|---|---|---|
| BR-001 Обучение системы | UC-01, UC-11 | Обучение на документах | Качество обучения и накопление знаний
| BR-002 Текстовые запросы | UC-02, UC-12 | Обработка запросов | Скорость и точность ответов
| BR-003 Проектно-ориентированность | UC-03, UC-13 | Работа с проектами | Поиск аналогов и извлечение данных
| BR-005 OCR | UC-04, UC-14 | Извлечение текста | Качество и скорость обработки
| BR-006 Excel отчеты | UC-05, UC-15 | Формирование отчетов | Структурирование и форматирование
| BR-007 ГОСТы | UC-06, UC-16 | Нормативная база | Валидация и поиск по нормативам

Примечание: термины и сокращения см. в глоссарии SRS — раздел «1.3 Термины и сокращения» (`documentation/requirements/srs.md`).

### Термины и сокращения (кратко)
- OCR — распознавание текста; CER/WER — ошибки по символам/словам
- macro‑F1 — усреднённая метрика качества классификации
- Валидность — % записей, прошедших JSON Schema без ошибок
- DWG/DXF — чертежи; ARP/GSFX — сметные форматы
- **Учитель системы** — роль для загрузки документов и обучения
- **Пользователь системы** — роль для выполнения текстовых запросов
- **Проектно-ориентированность** — привязка данных к проектам
- **Текстовые запросы** — естественные запросы на русском языке
- **Поиск аналогов** — поиск похожих проектов

## Таблица Use Cases (ядро интеллектуальной системы)

**Формат**: ID | Название | Краткое описание | Акторы | Предусловия | Основной сценарий | Ожидаемый результат

---

### UC-01 | Обучение системы на документах
**Краткое описание**: Учитель загружает проверенные документы для обучения системы  
**Акторы**: Учитель системы, Система обучения  
**Предусловия**: Документы проверены и готовы для обучения  
**Основной сценарий**: Загрузка документа → анализ содержимого → извлечение знаний → сохранение в базе знаний → валидация обучения  
**Ожидаемый результат**: Система изучила документ и может отвечать на вопросы по нему

- Acceptance: см. `documentation/requirements/metrics.md` (качество обучения, скорость обработки)
- Traceability: BR-001 ↔ SRS FR‑LEARN‑01..04

---

### UC-02 | Обработка текстовых запросов
**Краткое описание**: Пользователь делает текстовый запрос и получает структурированный ответ  
**Акторы**: Пользователь системы, Система обработки запросов  
**Предусловия**: Система обучена на документах  
**Основной сценарий**: Ввод текстового запроса → анализ запроса → поиск в базе знаний → формирование ответа → вывод результата  
**Ожидаемый результат**: Структурированный ответ на запрос пользователя

- Acceptance: см. `documentation/requirements/metrics.md` (точность ответов, скорость обработки)
- Traceability: BR-002 ↔ SRS FR‑QUERY‑01..04

---

### UC-03 | Проектно-ориентированная работа
**Краткое описание**: Система работает с данными в контексте конкретных проектов  
**Акторы**: Пользователь системы, Система проектов  
**Предусловия**: Данные привязаны к проектам  
**Основной сценарий**: Выбор проекта → запрос данных по проекту → извлечение данных → формирование отчета  
**Ожидаемый результат**: Данные извлечены в контексте проекта

- Acceptance: см. `documentation/requirements/metrics.md` (точность привязки к проектам)
- Traceability: BR-003 ↔ SRS FR‑PROJECT‑01..04

---

### UC-04 | Извлечение данных из документов (OCR)
**Краткое описание**: Система извлекает текст и данные из документов для обучения  
**Акторы**: Система OCR, Документы  
**Предусловия**: Документы загружены в систему  
**Основной сценарий**: Определение типа документа → OCR обработка → извлечение текста → структурирование данных  
**Ожидаемый результат**: Извлеченные данные готовы для обучения системы

- Acceptance: см. `documentation/requirements/metrics.md` (CER/WER, время обработки)
- Traceability: BR-005 ↔ SRS FR‑OCR‑01..04

---

### UC-05 | Формирование отчетов в Excel
**Краткое описание**: Система создает структурированные отчеты в Excel по запросам  
**Акторы**: Пользователь системы, Система отчетов  
**Предусловия**: Данные извлечены и структурированы  
**Основной сценарий**: Запрос отчета → извлечение данных → структурирование → форматирование → создание Excel файла  
**Ожидаемый результат**: Готовый отчет в Excel формате

- Acceptance: см. `documentation/requirements/metrics.md` (качество форматирования, скорость создания)
- Traceability: BR-006 ↔ SRS FR‑EXCEL‑01..04

---

### UC-06 | Работа с нормативной базой (ГОСТы)
**Краткое описание**: Система изучает и использует нормативные документы для валидации  
**Акторы**: Система нормативов, Нормативные документы  
**Предусловия**: Нормативные документы загружены в систему  
**Основной сценарий**: Загрузка ГОСТов → анализ нормативов → создание базы знаний → валидация данных  
**Ожидаемый результат**: Система использует нормативную базу для валидации

- Acceptance: см. `documentation/requirements/metrics.md` (полнота нормативной базы)
- Traceability: BR-007 ↔ SRS FR‑NORMS‑01..04

## Дополнительные Use Cases (на основе интервью)

### UC-11 | Поиск аналогов по проектам
**Краткое описание**: Пользователь ищет похожие проекты для извлечения данных  
**Акторы**: Пользователь системы, Система поиска аналогов  
**Предусловия**: Система обучена на проектных документах  
**Основной сценарий**: Запрос "где у нас есть козырьки" → поиск в базе знаний → анализ похожих проектов → извлечение данных → формирование отчета  
**Ожидаемый результат**: Список похожих проектов с извлеченными данными

### UC-12 | Извлечение ведомостей объемов работ
**Краткое описание**: Система извлекает ведомости объемов работ из чертежей по проекту  
**Акторы**: Пользователь системы, Система извлечения данных  
**Предусловия**: Чертежи проекта загружены в систему  
**Основной сценарий**: Запрос ведомости по проекту → анализ чертежей → извлечение объемов → структурирование → создание Excel отчета  
**Ожидаемый результат**: Ведомость объемов работ в Excel формате

### UC-13 | Формирование комплектов рабочей документации
**Краткое описание**: Система формирует полные комплекты документации по проекту  
**Акторы**: Пользователь системы, Система комплектования  
**Предусловия**: Документы проекта загружены в систему  
**Основной сценарий**: Запрос комплекта документации → анализ проекта → сбор документов → формирование списка → создание отчета  
**Ожидаемый результат**: Полный список документов проекта

### UC-14 | Обработка спецификации анкерных блоков
**Краткое описание**: Автоматическое извлечение и структурирование данных из спецификации металлических конструкций  
**Акторы**: Учитель системы, Система обработки спецификаций  
**Предусловия**: Загружена спецификация анкерных блоков в PDF формате  
**Основной сценарий**: Загрузка спецификации → OCR обработка → извлечение позиций → распознавание ГОСТов → структурирование → валидация масс  
**Ожидаемый результат**: Структурированная спецификация с техническими данными в Excel формате

### UC-15 | Обработка архитектурно-строительных чертежей
**Краткое описание**: Извлечение данных из архитектурно-строительных чертежей (АР)  
**Акторы**: Учитель системы, Система обработки чертежей  
**Предусловия**: Загружены архитектурно-строительные чертежи в PDF формате  
**Основной сценарий**: Загрузка чертежей → OCR обработка → извлечение размеров → распознавание обозначений → анализ спецификаций  
**Ожидаемый результат**: Структурированные данные чертежей с параметрами и спецификациями

### UC-16 | Работа с нормативными документами
**Краткое описание**: Система изучает и использует ГОСТы для валидации данных  
**Акторы**: Учитель системы, Система нормативов  
**Предусловия**: Нормативные документы загружены в систему  
**Основной сценарий**: Загрузка ГОСТов → анализ нормативов → создание базы знаний → валидация данных → поиск по нормативам  
**Ожидаемый результат**: Система использует нормативную базу для валидации и поиска
