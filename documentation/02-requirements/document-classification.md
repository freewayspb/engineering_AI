## 1. Проектно-ориентированная классификация документов

### 1.1 Концепция проектно-ориентированной работы
Все документы в системе AI GOST привязаны к конкретным проектам заказчика. Система работает в контексте проектов, позволяя:
- Извлекать данные по конкретному проекту
- Искать аналоги по проектам ("где у нас есть козырьки")
- Формировать комплекты рабочей документации по проектам
- Извлекать ведомости объемов работ по проектам

### 1.2 Примеры проектов заказчика
- **Подстанция беговая** - проект подстанции
- **Козырек** - проект козырька
- **И другие проекты** - по мере поступления от заказчика

### 1.3 Проектная структура данных
Каждый документ содержит метаданные:
- **Проект**: название проекта (подстанция, козырек и т.д.)
- **Тип документа**: чертеж, спецификация, смета, нормативный документ
- **Статус**: для обучения, проверенный, архивный
- **Дата загрузки**: когда документ был загружен в систему
- **Учитель**: кто загрузил документ для обучения

## 3. Типы документов по содержанию

### 3.1 Техническая документация
- **Проектная документация**: чертежи, схемы, планы
- **Технические задания**: требования к проектам
- **Нормативная документация**: ГОСТы, СНиПы, регламенты
- **Топографические съемки**: геодезические данные

### 3.2 Финансовая документация
- **Сметы**: расчеты стоимости работ и материалов
- **Финансовые отчеты**: бухгалтерская отчетность
- **Платежные документы**: счета, накладные, акты
- **Банковские документы**: выписки, справки

### 3.3 Управленческая документация
- **Корреспонденция**: письма, уведомления
- **Отчеты**: операционные и аналитические отчеты
- **Договоры**: контракты, соглашения
- **Регламенты**: процедуры и процессы

### 3.4 Нормативная документация (для обучения системы)
- **ГОСТы**: государственные стандарты
- **СНиПы**: строительные нормы и правила
- **Регламенты**: отраслевые регламенты
- **Стандарты**: корпоративные стандарты

## 4. Пользовательские сценарии (Use Cases)

### 4.1 Сценарий 1: Обучение системы на документах
**Описание**: Учитель загружает проверенные документы для обучения системы

#### Входные данные:
- Проверенные документы проекта (PDF, DWG, DOCX)
- Нормативные документы (ГОСТы, регламенты)
- Технические чертежи и спецификации
- Сметная документация

#### Процесс:
1. Учитель загружает документ в систему
2. Система определяет тип документа и проект
3. OCR обработка текстового содержимого
4. Извлечение знаний и структурирование данных
5. Сохранение в базе знаний системы
6. Валидация качества обучения

#### Выходные данные:
- Обновленная база знаний системы
- Метаданные документа (проект, тип, дата загрузки)
- Отчет о качестве обучения
- Статус готовности к ответам на запросы

#### Ожидаемый результат:
- Система изучила документ и может отвечать на вопросы по нему
- Документ привязан к проекту
- Знания накоплены в базе знаний

### 4.2 Сценарий 2: Обработка текстовых запросов
**Описание**: Пользователь делает текстовый запрос и получает структурированный ответ

#### Входные данные:
- Текстовый запрос пользователя на русском языке
- Контекст проекта (если указан)
- База знаний системы

#### Процесс:
1. Пользователь вводит текстовый запрос
2. Система анализирует запрос и понимает контекст
3. Поиск релевантной информации в базе знаний
4. Извлечение данных по запросу
5. Формирование структурированного ответа
6. Создание Excel отчета (если требуется)

#### Выходные данные:
- Структурированный ответ на запрос
- Excel файл с данными (если требуется)
- Список источников информации
- Метаданные поиска

#### Ожидаемый результат:
- Пользователь получил нужную информацию
- Данные структурированы и готовы к использованию
- Время обработки запроса не превышает 5 секунд

### 4.3 Сценарий 3: Поиск аналогов по проектам
**Описание**: Пользователь ищет похожие проекты для извлечения данных

#### Входные данные:
- Запрос типа "где у нас есть козырьки"
- База знаний по проектам
- Критерии поиска аналогов

#### Процесс:
1. Пользователь делает запрос на поиск аналогов
2. Система анализирует критерии поиска
3. Поиск похожих проектов в базе знаний
4. Анализ соответствия критериям
5. Извлечение данных по найденным проектам
6. Формирование отчета с аналогами

#### Выходные данные:
- Список похожих проектов
- Извлеченные данные по каждому проекту
- Excel файл с результатами поиска
- Анализ соответствия критериям

#### Ожидаемый результат:
- Найдены похожие проекты
- Извлечены нужные данные
- Сформирован отчет в Excel

### 4.4 Сценарий 4: Извлечение ведомостей объемов работ
**Описание**: Система извлекает ведомости объемов работ из чертежей по проекту

#### Входные данные:
- Чертежи проекта (PDF, DWG)
- Спецификации материалов
- Нормативные требования

#### Процесс:
1. Пользователь запрашивает ведомость по проекту
2. Система анализирует чертежи проекта
3. Извлечение объемов работ из чертежей
4. Анализ спецификаций материалов
5. Валидация по нормативным требованиям
6. Формирование ведомости в Excel

#### Выходные данные:
- Ведомость объемов работ в Excel
- Список материалов и их количества
- Нормативные ссылки
- Контрольные суммы

#### Ожидаемый результат:
- Полная ведомость объемов работ
- Данные готовы для использования в проекте
- Валидация выполнена по нормативам

## 6. Типы пользователей и уровни доступа

### 6.1 Ролевая модель (упрощенная)

| Роль | Описание | Доступ к информации | Права |
|------|----------|-------------------|-------|
| **Учитель системы** | Специалист по загрузке документов | Все типы документов для обучения | Загрузка документов, обучение системы, валидация |
| **Пользователь системы** | Специалист по запросам | Данные по текстовым запросам | Выполнение запросов, получение данных, формирование отчетов |
| **Администратор системы** | ИТ-специалист | Все типы документов | Полный доступ для настройки и поддержки |

### 6.2 Матрица доступа

| Роль | Обучение системы | Текстовые запросы | Формирование отчетов | Администрирование |
|------|------------------|-------------------|---------------------|-------------------|
| **Учитель системы** | ✅ Полный доступ | ❌ Нет доступа | ❌ Нет доступа | ❌ Нет доступа |
| **Пользователь системы** | ❌ Нет доступа | ✅ Полный доступ | ✅ Полный доступ | ❌ Нет доступа |
| **Администратор системы** | ✅ Полный доступ | ✅ Полный доступ | ✅ Полный доступ | ✅ Полный доступ |

### 6.3 Уровни доступа

#### Уровень 1: Пользователь системы
- Выполнение текстовых запросов
- Получение данных по запросам
- Формирование отчетов в Excel
- Поиск аналогов по проектам

#### Уровень 2: Учитель системы
- Загрузка документов для обучения
- Обучение системы на документах
- Валидация качества обучения
- Управление базой знаний

#### Уровень 3: Администратор системы
- Настройка системы
- Управление пользователями
- Конфигурация процессов
- Мониторинг работы системы

## 7. Интеграции с внешними системами

- **Отчетность**: формирование отчетов
- **Нормативные базы**: обновление ГОСТов и регламентов

### 7.3 Банковские системы
- **Платежные поручения**: автоматическое формирование
- **Выписки**: импорт и обработка
- **Справки**: генерация по запросу
