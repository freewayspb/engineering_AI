## Трассировка BR ↔ UC

| BR (из BRD) | UC | Назначение | Примечание |
|---|---|---|---|
| BR-001 Обучение системы | UC-01, UC-11 | Обучение на документах | Качество обучения и накопление знаний
| BR-002 Текстовые запросы | UC-02, UC-12 | Обработка запросов | Скорость и точность ответов
| BR-003 Проектно-ориентированность | UC-03, UC-13 | Работа с проектами | Поиск аналогов и извлечение данных
| BR-005 OCR | UC-04, UC-14 | Извлечение текста | Качество и скорость обработки
| BR-006 Excel отчеты | UC-05, UC-15 | Формирование отчетов | Структурирование и форматирование
| BR-007 ГОСТы | UC-06, UC-16 | Нормативная база | Валидация и поиск по нормативам

Примечание: термины и сокращения см. в глоссарии SRS — раздел «1.3 Термины и сокращения» (`technical-requirements.md`).

### Термины и сокращения (кратко)
- OCR — распознавание текста; CER/WER — ошибки по символам/словам
- macro‑F1 — усреднённая метрика качества классификации
- Валидность — % записей, прошедших JSON Schema без ошибок
- DWG/DXF — чертежи; ARP/GSFX — сметные форматы
- **Учитель системы** — роль для загрузки документов и обучения
- **Пользователь системы** — роль для выполнения текстовых запросов
- **Проектно-ориентированность** — привязка данных к проектам
- **Текстовые запросы** — естественные запросы на русском языке
- **Поиск аналогов** — поиск похожих проектов

### UC-01 | Обучение системы на документах
**Краткое описание**: Учитель загружает проверенные документы для обучения системы  
**Акторы**: Учитель системы, Система обучения  
**Предусловия**: Документы проверены и готовы для обучения  
**Основной сценарий**: Загрузка документа → анализ содержимого → извлечение знаний → сохранение в базе знаний → валидация обучения  
**Ожидаемый результат**: Система изучила документ и может отвечать на вопросы по нему

- Acceptance: см. `metrics.md` (качество обучения, скорость обработки)
- Traceability: BR-001 ↔ SRS FR‑LEARN‑01..04

---

### UC-02 | Обработка текстовых запросов
**Краткое описание**: Пользователь делает текстовый запрос и получает структурированный ответ  
**Акторы**: Пользователь системы, Система обработки запросов  
**Предусловия**: Система обучена на документах  
**Основной сценарий**: Ввод текстового запроса → анализ запроса → поиск в базе знаний → формирование ответа → вывод результата  
**Ожидаемый результат**: Структурированный ответ на запрос пользователя

- Acceptance: см. `metrics.md` (точность ответов, скорость обработки)
- Traceability: BR-002 ↔ SRS FR‑QUERY‑01..04

---

### UC-03 | Проектно-ориентированная работа
**Краткое описание**: Система работает с данными в контексте конкретных проектов  
**Акторы**: Пользователь системы, Система проектов  
**Предусловия**: Данные привязаны к проектам  
**Основной сценарий**: Выбор проекта → запрос данных по проекту → извлечение данных → формирование отчета  
**Ожидаемый результат**: Данные извлечены в контексте проекта

- Acceptance: см. `metrics.md` (точность привязки к проектам)
- Traceability: BR-003 ↔ SRS FR‑PROJECT‑01..04

---

### UC-04 | Извлечение данных из документов (OCR)
**Краткое описание**: Система извлекает текст и данные из документов для обучения  
**Акторы**: Система OCR, Документы  
**Предусловия**: Документы загружены в систему  
**Основной сценарий**: Определение типа документа (mc_pdf/scan_pdf) → (для scan_pdf: предобработка) → OCR обработка → извлечение текста и реквизитов штампа → структурирование данных  
**Ожидаемый результат**: Извлеченные данные готовы для обучения системы

- Acceptance: см. `metrics.md` (CER/WER, время обработки)
- Traceability: BR-005 ↔ SRS FR‑OCR‑01..04

---

### UC-05 | Формирование отчетов в Excel
**Краткое описание**: Система создает структурированные отчеты в Excel по запросам  
**Акторы**: Пользователь системы, Система отчетов  
**Предусловия**: Данные извлечены и структурированы  
**Основной сценарий**: Запрос отчета → извлечение данных → структурирование → выбор шаблона подразделения (ПТО/Сметный отдел) → форматирование → создание Excel файла  
**Ожидаемый результат**: Готовый отчет в Excel формате (соответствует выбранному шаблону подразделения)

- Acceptance: см. `metrics.md` (качество форматирования, скорость создания)
- Traceability: BR-006 ↔ SRS FR‑EXCEL‑01..04

---

### UC-06 | Работа с нормативной базой (ГОСТы)
**Краткое описание**: Система изучает и использует нормативные документы для валидации  
**Акторы**: Система нормативов, Нормативные документы  
**Предусловия**: Нормативные документы загружены в систему  
**Основной сценарий**: Загрузка ГОСТов → анализ нормативов → создание базы знаний → валидация данных  
**Ожидаемый результат**: Система использует нормативную базу для валидации

- Acceptance: см. `metrics.md` (полнота нормативной базы)
- Traceability: BR-007 ↔ SRS FR‑NORMS‑01..04

## Дополнительные Use Cases (на основе интервью)

### UC-04b | Estimate Parsing (ARP/GSFX/XML/XLS)

**Основной сценарий**: Определение формата → парсинг разделов/позиций/формул → привязка к шифру РД → валидация totals/единиц → структурирование и экспорт в XLSX

- [ ] В `titleBlock` заполнены ключевые поля (title, cipher, sheetNumber, documentDate, revisionNumber)

- [ ] Артефакты сохранены в хранилище (metadata JSON)

#### UC-22 | PDF (МЧ): Извлечение табличного содержания спецификации
- **Акторы**: Система парсинга таблиц, Документ PDF (МЧ)

- **Выход/Результат**: Таблица спецификации (JSON/CSV) + XLS черновик
- **Роль/Подразделение**: Инженер / ПТО
- **Цель**: Сводная спецификация; унификация; бюджетирование
 
Acceptance (без метрик):
- [ ] Выделены корректные границы таблиц
- [ ] Экспортирована таблица (JSON/CSV) и черновой XLS
- [ ] Числовые поля распознаны корректно (типы/форматы)
