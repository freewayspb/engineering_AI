## Реструктуризация проекта AI GOST (без изменения содержимого файлов)

**Цель**  
Улучшить навигацию в проекте, приведя файловую структуру к логичной иерархии, не изменяя содержимое документов.

**Жёсткие ограничения**
- Не редактируй текст, форматирование, метаданные и содержимое существующих файлов.
- Ничего не удаляй.
- Все текущие межфайловые ссылки должны остаться рабочими. Разрешено корректировать только относительные пути в ссылках, если они должны вести на новое расположение.

**Разрешено**
- Создавать и переименовывать папки.
- Перемещать файлы между папками.
- Переименовывать файлы (с сохранением расширения) для читабельности.
- Создавать новые `README.md` в папках для навигации (их содержимое пишем с нуля).

---

## Шаги

1. **Карта текущего состояния**  
   Проанализируй существующую структуру `documentation/` и выдели ключевые тематические блоки.

2. **Создание новой иерархии**  
   Сформируй нумерованные каталоги для логической сортировки:

   ```
   documentation/
   ├── 01-project-overview
   ├── 02-requirements
   ├── 03-prototypes
   ├── 04-project-management
   ├── 05-interviews
   ├── 06-assets
   └── 07-archive
   ```

3. **Перемещение и переименование файлов**  
   Расположи файлы в новых папках. Рекомендуемые переименования:

   - `vision.md` → `project-vision.md`
   - `brd.md` → `business-requirements.md`
   - `srs.md` → `technical-requirements.md`
   - `history.md` → `project-history.md`

   Пример распределения по каталогам:

   - `01-project-overview`: `README.md`, `project-vision.md`, `project-history.md`
   - `02-requirements`: `business-requirements.md`, `technical-requirements.md`, `use-cases.md`, `document-classification.md`, `metrics.md`, `schemas/`
   - `03-prototypes`: `README.md`, `prototype-1-ocr.md`, `prototype-2-structuring.md`, `prototype-3-classification.md`
   - `04-project-management`: `work-plan.md`, `customer-report.md`, `contracts/`
   - `05-interviews`: `README.md`, `checklist.md`, `questions.md`, `templates/`, `build/`, прочие материалы интервью
   - `06-assets`: `README.md`, `audio/`, `images/`, `input-documents/`
   - `07-archive`: `README.md`, `deprecated/`, `migration-log.md`

4. **README для каждой папки**  
   В каждом разделе создай `README.md` со следующими блоками:
   - Описание назначения папки.
   - Таблица с файлами, краткими описаниями и статусом.
   - Ссылки на родительскую и связанные папки.

5. **Обновление корневого обзора**  
   В `documentation/README.md` опиши новую структуру, добавь ссылки на все разделы и укажи статусы (например, Draft / In progress / Approved).

6. **Проверка ссылок**  
   После перемещений проверь все относительные ссылки. При необходимости обнови только пути (оставив текст ссылок неизменным).

7. **Контрольный список завершения**
   - [ ] Все файлы присутствуют в новой структуре, ничего не потеряно.
   - [ ] Все относительные ссылки внутри документации работают.
   - [ ] В каждой папке есть `README.md` с навигацией.
   - [ ] `documentation/README.md` отражает новую иерархию.
   - [ ] Папки сортируются по числовым префиксам.

---

## Ожидаемый результат

- Единообразная, нумерованная структура каталогов.
- Понятная навигация благодаря индексам в каждой папке.
- Все документы доступны и содержимое неизменно.
- Межфайловые ссылки продолжают работать после реструктуризации.
